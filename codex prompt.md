# 项目骨架
```
你现在是一个资深 Android 开发工程师，请帮我从零创建一个 Android App 项目骨架，使用 Kotlin + Jetpack Compose + Material3。

项目名称：PocketLedger（中文名：“口袋账本”）

【技术要求】
- Kotlin
- Jetpack Compose UI
- Material3 设计体系
- 使用 MVVM 架构（ViewModel）
- 使用 Room 作为本地数据库（后续用）
- Navigation Compose 做页面导航

【整体目标】
先搭建完整项目结构 + 页面骨架，不需要实现具体业务逻辑。

【需要完成的内容】

1. 项目结构设计：
- data（数据层）
- ui（界面层）
- viewmodel（状态管理）
- navigation（导航）
- theme（主题）

2. 底部导航栏（Bottom Navigation）包含5个页面：
- HomeScreen（首页）
- RecordScreen（记录页）
- CalendarScreen（日历页）
- SavingsScreen（存钱小荷包）
- SettingsScreen（设置页）

3. 每个页面先用占位UI：
- 显示页面标题（居中）
- 简单卡片结构（Material3 Card）

4. 首页（HomeScreen）特殊要求：
- 顶部一个大卡片（深色渐变风格）
- 显示：
  - 当前结余（大字体）
  - 今日已花
  - 本月总支出
（先写死假数据）

5. 主题设计（非常重要）：
- 支持浅色 + 深色模式
- 主色：紫色（偏蓝紫）
- 使用 Material3 动态配色结构
- 卡片圆角：16dp以上
- 整体风格偏现代、简洁

6. Navigation：
- 使用 Navigation Compose
- 底部导航切换页面

7. 提供：
- 完整代码结构
- 关键文件代码
- 每个模块说明

【注意】
先不要实现数据库和业务逻辑，只搭UI骨架和结构。
代码要规范、可运行。
```

# 记一笔功能
```
请继续完善我当前的 Android App 项目。技术栈保持不变：Kotlin + Jetpack Compose + Material3 + Navigation Compose。

这次要实现“记一笔页面（核心功能）”，并调整首页交互逻辑。

【产品需求】
1. 首页不仅展示概要信息，还要支持“快速记一笔”。
2. 首页底部导航栏上方有一个悬浮加号按钮（Floating Action Button）。
3. 点击这个加号按钮后，跳转到“记一笔页面”。
4. “记录页”只负责查看历史记录，不承担新增记录入口。

【页面与交互要求】

一、HomeScreen（首页）
- 保留当前首页顶部大卡片，用于展示：
  - 当前结余
  - 今日已花
  - 本月总支出
- 页面底部仍然保留 Bottom Navigation
- 在页面底部中央、导航栏上方放置一个悬浮加号按钮（FAB）
- FAB 风格要精致，和整体紫蓝色主题一致
- 点击 FAB 后导航到 AddRecordScreen

二、AddRecordScreen（记一笔页面）
这是本次开发重点，请认真实现 UI 骨架和基础交互。

页面内容包括：
1. 顶部栏
- 返回按钮
- 页面标题：“记一笔”

2. 输入区域
- 消费内容输入框
  - label: 内容
  - placeholder: 例如“早饭”“地铁”“打印资料”

- 金额输入区域
  - 不是普通输入框，而是“计算器样式数字键盘”
  - 页面上方显示当前输入金额
  - 支持数字 0-9、小数点、删除
  - 先不用做复杂四则运算
  - 默认金额显示为 0

3. 类型选择
- 使用单选样式或 segmented button
- 三种类型：
  - 支出
  - 退款
  - 收入
- 默认选中“支出”

4. 日期时间
- 默认自动带出当前日期和当前时间
- 先只做展示和基础点击样式
- 后续再接真正的日期选择器和时间选择器
- 当前阶段允许写成可点击卡片：“日期：2026-04-01”“时间：08:35”

5. 自动分类展示
- 根据内容关键词自动分类
- 当前先做一个简单函数：
  - 包含“饭”“早餐”“午饭”“晚饭”“奶茶”“咖啡”"瑞幸" -> 餐饮
  - 包含“地铁”“公交”“打车”“电动车” -> 交通
  - 包含“教材”“打印”“文具”“GPT” -> 学习
  - 包含“电影”“游戏”“攀岩”“旅游”“火车”“高铁”“飞机”“门票” -> 娱乐
  - 包含“药”“医院” -> 医疗
  - 包含“衣”“裤”“鞋” -> 衣服
  - 包含“纸”“牙膏”“洗发水”“卫生巾” -> 日用品
  - 否则 -> 其他
- 在页面中展示“自动分类结果：xxx”
- 注意：分类只用于展示，不参与金额计算

6. 保存按钮
- 页面底部有一个主按钮：“保存”
- 当前点击后先不接数据库
- 暂时打印日志或弹 Toast：显示用户输入的内容、金额、类型、分类、日期时间
- 代码结构要为后续接 Room 数据库预留好

三、RecordScreen（记录页）
- 明确改为“仅查看记录”
- 删除或移除原本新增记录入口
- 暂时使用假数据列表展示几条记录
- 每条记录显示：
  - 内容
  - 金额
  - 类型
  - 分类
  - 时间

【代码结构要求】
1. 新增 AddRecordScreen.kt
2. 新增 AddRecordUiState 数据类
3. 新增 AddRecordViewModel
4. 导航中注册 AddRecordScreen 路由
5. 首页 FAB 触发导航
6. 分类逻辑单独提取成一个 utils 或 helper 函数，便于后续替换

【UI风格要求】
- 延续当前项目的现代化卡片风格
- 浅色背景 + 局部深色强调
- 圆角偏大
- 间距舒展
- 输入页不要做得像后台表单，要更像精致消费记录 App

【请同时输出】
1. 关键代码文件
2. 每个文件的作用说明
3. HomeScreen 如何接入 FAB
4. AddRecordScreen 的 Compose 结构说明
5. 我作为初学者需要重点看懂哪几部分
```
