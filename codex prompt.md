# 梳理需求
1. 产品目标。
- [ ] 用户每消费一笔，都能快速录入。
- [ ] 系统自动记录时间、内容、金额、类型，并准确计算结余与统计结果。
- [ ] 系统以“日常流水小荷包”和“存钱小荷包”两个账户分离消费与储蓄。
- [ ] 整个 App 必须保证金额计算是确定性的，智能功能只能参与分类或展示，不能参与财务计算。

2. 账户体系需求。
整个 App 采用双账户模型。

- [ ] 第一个账户是“日常流水小荷包”。它是每月独立结算的主消费账户，用来记录支出、退款和收入。
	- [ ] 这个账户每月 1 号自动发放 2000 元生活费，作为该月预算基线。
	- [ ] 用户在日常生活中输入的绝大多数记录，都进入这个账户。
	- [ ] 这个账户允许结余为负数，也就是允许超支，但超支只作为本月历史存在，不结转到下个月。
- [ ] 第二个账户是“存钱小荷包”。它是独立储蓄账户，用于存放每月结余和用户主动储蓄的钱。
	- [ ] 它只能进不能出，不允许转回日常流水账户。

3. 日常流水小荷包的记录类型规则。

- [ ] 日常流水小荷包必须支持三种记录类型：支出、退款、收入。
- [ ] 支出表示正常花钱，比如吃饭、交通、购物，会增加本月支出，并减少当前结余。
- [ ] 退款不属于收入，它的本质是对已有支出的冲减。比如本月支出 1200，退款 200，则本月实际支出应为 1000。退款会增加当前结余，但不会记入“收入”。
- [ ] 收入表示额外入账，比如兼职费、临时转账等。收入会增加当前结余，但不影响“本月总支出”的统计口径。

4. 下面是月度结算规则。

- [ ] 每月 1 号，日常流水小荷包自动重置，并自动发放 2000 元。新月份的预算从 2000 重新开始，不受上个月超支影响。
- [ ] 每月月底，系统对该月日常流水小荷包进行结算。如果月底结余为正数，则自动转入“存钱小荷包”。如果月底结余为负数，则不报错，也不从存钱小荷包扣减，只作为该月历史记录保留。
这意味着每个月都是独立预算周期。超支不背到下个月；结余才会沉淀为储蓄。

5. 下面是记一笔功能需求。

- [ ] 记一笔是整个 App 的最高频核心功能。它应该从首页直接触发，不需要用户先切到记录页。
- [ ] 首页底部导航栏上方有一个中央悬浮加号按钮，点击后进入独立的“记一笔”页面。
- [ ] 记一笔页面需要包含以下内容：顶部返回栏；内容输入框；金额展示区；计算器样式数字键盘；记录类型选择器；日期展示；时间展示；自动分类结果；保存按钮。
- [ ] 内容输入框用于填写“早饭”“地铁”“打印资料”这样的自然语言描述。
- [ ] 金额输入采用计算器样式键盘，而不是普通系统输入框。需要支持数字 0 到 9、小数点和删除。默认金额显示为 0。当前阶段先不支持复杂四则运算，但要保证金额输入逻辑清晰，比如初始为 0 时输入数字应替换，已有小数点时不能重复输入，删除到空时应回到 0。
- [ ] 类型选择需要支持支出、退款、收入三种，默认选中支出。
- [ ] 日期和时间默认自动带出当前时间，且精确到分钟。当前阶段可以先做展示型卡片，后续再接入真正的日期选择器和时间选择器。
- [ ] 自动分类功能必须有，但分类不能由你手动选，也不能让 AI 介入金额计算。你最终选择的是“自动关键词分类”。系统根据内容字段里的关键词，自动将记录归类为餐饮、交通、学习、娱乐、医疗、衣服、日用品，无法匹配时归为其他。这个分类结果只参与展示和统计，不参与金额运算。
- [ ] 保存按钮当前的最终目标是将这条记录保存到本地数据库，并驱动首页和记录页联动更新。

6. 首页需求。

- [ ] 首页最核心的信息有三项：当前结余、今日已花、本月总支出。其中“当前结余”要作为最显眼的大数字放在顶部大卡片中。

7. 记录页需求。

 - [ ] 记录页是专门查看历史流水的页面，不再承担新增记录入口。它的职责就是让你按时间查看已保存记录。

8. 日历页需求。

它不仅是日期选择器，更是消费可视化界面。页面中应有可直接选择日期的月历视图，风格类似![[Pasted image 20260401081602.png|212]]

- [ ] 点击某一天后，页面应先显示该日总支出，再展示该日的全部记录列表。也就是先给出“结论”，再展示明细。

你明确希望 UI 风格接近成熟商业产品，而不是简单的学生练手项目。整体风格应当是现代、精致、卡片式、留白充分，主色调偏紫蓝。你不想要纯深色或纯浅色，而是希望“深浅搭配”。

下面是数据存储需求。

当前阶段你选择的是本地优先，不做云同步。也就是说，所有记录首先保存在手机本地数据库中，保证离线可用、成本低、结构稳定。未来可以再考虑云同步，但不作为第一阶段必须功能。

下面是智能功能边界。

这个项目允许有“智能”，但智能必须严格受限。金额计算、结余计算、月结规则、存钱转入，这些财务逻辑必须由固定规则驱动，不能交给 AI 或模糊判断。自动分类可以用关键词匹配，未来也可以升级为 AI 分类，但 AI 只负责分类，不得参与金钱运算。
